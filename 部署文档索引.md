### 5. 服务器环境一键安装脚本

**文件**: `server-setup.sh`

**功能**：
- 自动安装所有必要软件
- 配置数据库和用户
- 设置防火墙规则

**使用方法**：
```bash
# 方法1：上传脚本后执行
chmod +x server-setup.sh
sudo bash server-setup.sh

# 方法2：腾讯云用户可直接使用
# 打开 腾讯云一键安装命令.txt，复制命令执行
```

### 6. 快速部署脚本

**文件**: `deploy.sh`

**功能**：
- 自动拉取代码
- 编译后端
- 构建前端
- 重启服务

**使用方法**：
```bash
# 部署全部
./deploy.sh all

# 只部署后端
./deploy.sh backend

# 只部署前端
./deploy.sh frontend
```

### 7. 生产环境配置示例

**文件**: `backend/src/main/resources/application-prod.yml`

**内容**：
- 生产环境配置模板
- 数据库连接配置
- 日志配置
- 性能优化配置

**使用方法**：
```bash
java -jar -Dspring.profiles.active=prod app.jar
```

## 🚀 快速开始

### 腾讯云用户（推荐）

1. **购买腾讯云服务器**
   - 参考 `TENCENT_CLOUD_SETUP.md` 第1章

2. **配置安全组**
   - 开放22, 80, 443端口
   - 参考 `TENCENT_CLOUD_SETUP.md` 第2章

3. **一键安装环境**
   - 打开 `腾讯云一键安装命令.txt`
   - 复制命令到腾讯云Web Shell执行
   - 或参考 `TENCENT_CLOUD_SETUP.md` 详细步骤

4. **部署项目**
   - 参考 `DEPLOYMENT_GUIDE.md` 继续

### 首次部署（通用）

1. **阅读完整教程**
   - 打开 `DEPLOYMENT_GUIDE.md`
   - 按照步骤1-12执行

2. **使用一键安装脚本**（可选）
   ```bash
   sudo bash server-setup.sh
   ```

3. **配置项目**
   - 修改 `application.yml`
   - 配置数据库连接
   - 设置API密钥

4. **部署项目**
   ```bash
   ./deploy.sh all
   ```

### 更新部署

```bash
# 使用部署脚本
./deploy.sh all

# 或手动执行
cd backend && git pull && mvn package
cd frontend && git pull && npm run build
systemctl restart tibet-backend
systemctl reload nginx
```

## 📋 部署检查清单

部署前请确认：

- [ ] 已购买服务器
- [ ] 已配置安全组
- [ ] 已准备域名（可选）
- [ ] 已获取API密钥（豆包API等）
- [ ] 已阅读完整部署教程

部署后请验证：

- [ ] 后端服务运行正常（端口8080）
- [ ] 前端页面可以访问
- [ ] API接口正常响应
- [ ] 数据库连接正常
- [ ] SSL证书已配置（如使用HTTPS）
- [ ] 日志文件正常生成
- [ ] 定时任务正常运行

## 🔧 配置文件位置

| 配置文件 | 路径 | 说明 |
|---------|------|------|
| 后端配置 | `backend/src/main/resources/application.yml` | 开发环境配置 |
| 生产配置 | `backend/src/main/resources/application-prod.yml` | 生产环境配置 |
| Nginx配置 | `/etc/nginx/sites-available/colorful-tibet` | Web服务器配置 |
| 系统服务 | `/etc/systemd/system/tibet-backend.service` | 后端服务配置 |

## 🆘 获取帮助

### 遇到问题？

1. **查看常见问题**
   - `DEPLOYMENT_GUIDE.md` 第9章

2. **查看日志**
   ```bash
   # 后端日志
   tail -f /opt/colorful-tibet/logs/backend.log
   
   # Nginx日志
   tail -f /var/log/nginx/colorful-tibet-error.log
   ```

3. **检查服务状态**
   ```bash
   systemctl status tibet-backend
   systemctl status nginx
   systemctl status mysql
   ```
